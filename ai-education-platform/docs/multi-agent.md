# 多智能体系统文档

## 1. 概述

本系统的核心创新在于**四个协作智能体**，它们通过共享数据层和协调机制，共同为学生提供个性化的学习支持。

## 2. 智能体详解

### 2.1 学情分析Agent (StudentAnalysisAgent)

**职责**：全面分析学生的学习状态，识别学习风险。

**输入数据**：
- 历次考试成绩
- 作业完成情况与得分
- 课堂签到记录
- 学习资源使用记录
- 课堂互动数据

**分析维度**：
```
1. 成绩趋势分析
   - 近N次成绩走势（上升/下降/波动）
   - 与班级平均分对比
   - 各科目成绩分布

2. 薄弱点识别
   - 错题知识点聚类
   - 高频错误题型
   - 概念理解偏差

3. 学习风格推断
   - 视觉型（偏好图表/视频）
   - 听觉型（偏好讲解/音频）
   - 实践型（偏好练习/实验）

4. 风险评估
   - 学业预警等级（安全/关注/警告/危险）
   - 预测期末成绩
   - 挂科风险概率
```

**输出格式**：
```json
{
  "risk_level": "warning",
  "weak_points": ["线性代数-矩阵运算", "概率论-条件概率"],
  "strong_points": ["微积分-导数"],
  "learning_style": "visual",
  "score_trend": "declining",
  "predicted_score": 68,
  "recommendations": ["建议重点复习矩阵运算", "推荐观看相关视频"]
}
```

### 2.2 资源生成Agent (ResourceGenerationAgent)

**职责**：基于学情分析结果，生成个性化的学习资源。

**输入数据**：
- 学生学习画像（来自学情分析Agent）
- 课程大纲和知识点体系
- 学生历史学习记录

**生成内容**：
```
1. 个性化练习题
   - 针对薄弱知识点的专项练习
   - 难度梯度设计（基础→进阶→挑战）
   - 题型多样化（选择/填空/解答）

2. 知识点解析
   - 用学生易懂的语言解释概念
   - 类比和实际案例
   - 常见错误分析

3. 学习计划
   - 每日学习任务安排
   - 复习时间分配
   - 阶段性目标设定

4. 资源推荐
   - 相关教学视频
   - 参考资料
   - 课程回放切片
```

**Prompt 设计示例**：
```
你是一位经验丰富的{course_name}教师。
学生{student_name}在以下知识点存在薄弱环节：
{weak_points}

请为该学生生成：
1. 5道针对性练习题（含答案和解析）
2. 这些知识点的简明讲解
3. 一周的学习计划

要求：
- 难度适中，循序渐进
- 语言通俗易懂
- 注重实际应用
```

### 2.3 教学优化Agent (TeachingOptimizationAgent)

**职责**：分析班级整体学习情况，为教师提供教学优化建议。

**输入数据**：
- 班级整体成绩分布
- 各知识点掌握情况
- 作业完成率和正确率
- 签到率和课堂互动

**分析输出**：
```
1. 班级画像
   - 成绩分布（优秀/良好/中等/及格/不及格占比）
   - 班级整体薄弱点
   - 学习积极性指标

2. 教学建议
   - 需要重点复习的知识点
   - 建议调整的教学进度
   - 推荐的教学方法

3. 课堂优化
   - 哪些内容需要重新讲解
   - 哪些内容可以加快进度
   - 课堂互动改进建议
```

### 2.4 考试分析Agent (ExamAnalysisAgent)

**职责**：深度分析考试数据，提供多维度的考试评估。

**输入数据**：
- 考试成绩列表
- 每题得分情况
- 题目知识点标签

**分析维度**：
```
1. 题目分析
   - 每题通过率
   - 区分度分析
   - 难度等级评估

2. 失分模式
   - 高频错题
   - 错题知识点聚类
   - 时间分配分析

3. 改进建议
   - 题目质量评估
   - 考试范围建议
   - 针对性复习方案
```

## 3. Agent 协作机制

### 3.1 数据流

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  考试/作业   │────▶│ 考试分析Agent│────▶│ 学情分析Agent│
│  数据入库    │     └─────────────┘     └──────┬──────┘
└─────────────┘                                │
                                               ▼
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│ 学生收到     │◀────│ 资源生成Agent│◀────│ 学习画像更新 │
│ 个性化方案   │     └─────────────┘     └─────────────┘
└─────────────┘
                                               ▲
┌─────────────┐     ┌─────────────┐           │
│ 教师调整     │◀────│ 教学优化Agent│───────────┘
│ 教学策略     │     └─────────────┘
└─────────────┘
```

### 3.2 触发机制

| 事件 | 触发的Agent | 说明 |
|------|------------|------|
| 学生提交作业 | 考试分析 → 学情分析 → 资源生成 | 作业批改后自动触发分析 |
| 考试成绩录入 | 考试分析 → 教学优化 | 全班成绩分析 |
| 学生查看资源页 | 资源生成 | 根据最新画像推荐资源 |
| 教师查看班级分析 | 教学优化 | 汇总班级数据 |
| 学期中期 | 学情分析 | 批量风险评估 |

### 3.3 协调策略

1. **串行依赖**：资源生成依赖学情分析的结果
2. **并行独立**：教学优化和学情分析可以并行执行
3. **结果缓存**：分析结果存入数据库，避免重复计算
4. **增量更新**：只分析变化的数据，提高效率

## 4. LLM 集成策略

### 4.1 Mock 模式（当前实现）

开发和演示阶段使用模拟响应，特点：
- 响应速度快，无需网络
- 内容质量可控，适合演示
- 支持所有分析场景

### 4.2 切换到真实 LLM

只需实现 `LLMService` 接口：

```python
class LLMService:
    async def analyze_text(self, prompt: str, context: dict) -> str:
        """调用LLM进行文本分析"""
        raise NotImplementedError
    
    async def generate_questions(self, topic: str, difficulty: int, count: int) -> list:
        """生成练习题"""
        raise NotImplementedError
    
    async def generate_explanation(self, question: str, answer: str) -> str:
        """生成题目解析"""
        raise NotImplementedError
    
    async def generate_study_plan(self, profile: dict, goals: list) -> dict:
        """生成学习计划"""
        raise NotImplementedError
```

### 4.3 支持的 LLM 服务商

| 服务商 | 接入方式 | 适用场景 |
|--------|---------|---------|
| OpenAI GPT | API Key | 通用场景，质量最高 |
| 百度文心一言 | API Key | 中文优化 |
| 讯飞星火 | API Key | 教育领域优化 |
| ChatGLM | 本地部署 | 数据隐私要求高 |
| LLaMA | 本地部署 | 完全离线 |

## 5. 性能优化

- **分析结果缓存**：相同条件的分析结果缓存24小时
- **批量处理**：学期初批量生成全班学习画像
- **异步执行**：长时间分析任务异步执行，前端轮询结果
- **增量更新**：只分析新增的考试/作业数据
